# V0.2L — Workspace Session Control: Archive Current + Start Fresh

## Purpose

This slice adds the minimum workspace-management surface needed for repeated validation work.

The build/test loop had accumulated one long-running workspace with many tabs, recovery events, diagnostics, and journal records. That history is useful, but it can also contaminate validation of new behavior.

This feature lets the Operator preserve the current workspace trace, then start a clean active workspace for the next validation slice.

## Scope

This is not full database/workspace switching yet.

It adds:

- Archive Current Workspace
- Archive + Start Fresh Workspace
- Copy Workspace Snapshot

It does not yet add:

- Workspace list UI
- Restore archived workspace
- Delete archived workspace
- Rename archived workspace
- Import/export files
- Full database abstraction

## Storage Keys

Active workspace:

```text
chromeFlowWorkspace
```

Archived workspaces:

```text
chromeFlowWorkspaceArchive
```

Developer diagnostics remain separate:

```text
chromeFlowDiagnostics
```

## Behavior

Archive Current Workspace:

- stores a copy of the active workspace in local Chrome storage
- keeps the active workspace unchanged
- records a developer diagnostic event

Archive + Start Fresh Workspace:

- archives the current workspace first
- creates a new empty active workspace
- keeps browser tabs open
- keeps Developer Diagnostics unchanged
- reloads the side panel into the clean workspace

Copy Workspace Snapshot:

- copies a JSON snapshot of the active workspace and archive summaries
- includes a warning that titles, URLs, notes, and system events may be included

## Archive Limit

The archive is capped at 20 workspace snapshots to reduce local storage risk.

This is an early safety cap, not the final memory/database policy.
