# V0.2O Patch — Recovery Group Restore

## Purpose

Validation confirmed that `Move Workspace Into New Window` now moves workspace tabs into a new Chrome window and recreates role-based Chrome groups correctly.

A small overlap remained in the Recovery Journal:

```text
Close Browser Tab
Re-add to Workspace
```

The recovered tab reopened and re-entered the workspace, but it was not automatically re-added to the Chrome group matching its saved role. The Operator had to click `Create Chrome Tab Groups` again.

## Runtime Placement

This patch adds a small companion module:

```text
src/sidepanel/recovery-group-restore.js
```

Current side panel load order:

```text
diagnostics.js
sidepanel.js
workspace-session-control.js
search-workspace-intake.js
recovery-group-restore.js
advanced-tab-controls.js
advanced-new-window-stability-v2.js
```

The native recovery function still owns re-add/reopen behavior. The helper waits for the native recovery action to complete, then restores the recovered tab to its role group.

## Behavior

When the Operator clicks `Re-add to Workspace`, the helper:

1. Reads the recovery event id from the clicked button.
2. Waits briefly for native `workspace_tab_readded` to complete.
3. Finds the newest matching `workspace_tab_readded` event.
4. Resolves the recovered workspace tab to its live browser tab.
5. Finds an existing grouped tab with the same role in the same window.
6. Adds the recovered tab to that Chrome group.
7. If no matching role group exists, creates a new group for the recovered tab.
8. Refreshes workspace tab metadata from live browser tabs.
9. Records `recovered_tab_group_restored`.
10. Records diagnostic action `recovery_group_restored`.

## Expected Evidence

A clean pass should show:

```text
workspace_tab_readded
recovered_tab_group_restored
recovery_group_restored
groupedTabs == totalTabs
ungroupedTabs: 0
pendingActionTraces: []
```

## Validation Checklist

1. Pull latest and reload the extension.
2. Start with several workspace tabs assigned to roles.
3. Create Chrome groups.
4. Move Workspace Into New Window.
5. Confirm groups persist.
6. Close one grouped browser tab through Chrome Flow.
7. In Recovery Journal, click `Re-add to Workspace`.
8. Confirm the tab reopens.
9. Confirm the tab is automatically re-added to the matching role group.
10. Copy workspace and diagnostic packets.

## Integration Note

This remains a helper module until the next deliberate native-controller rewrite pass. If validated, its behavior should eventually be merged into the native recovery function in `sidepanel.js`.
