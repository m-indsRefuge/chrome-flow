# V0.2N — Tab Management Closure Pass

## Purpose

V0.2N closes the deterministic tab-management phase before Chrome Flow moves toward the AI command contract.

The aim is not full UI redesign. The aim is to make the tab-control surface feel complete, observable, and safe enough that a later AI layer can propose deterministic tab actions against a stable base.

## Build Method Note

Chrome Flow is now being built with a phase-complete layer method:

```text
1. Complete a deterministic management layer.
2. Harden that layer through validation packets and local tests.
3. Draft the AI command contract against the stable deterministic layer.
4. Test AI proposals through the deterministic action surface.
5. Then move to the next management layer.
```

This differs from Lighthouse, where the early V0 prototype is being built more holistically because telemetry, diagnosis, permissioning, memory, and local model behavior are more tightly interdependent.

## Scope

V0.2N adds:

- Add Current Active Tab
- Select All Scanned Tabs
- Deselect All Scanned Tabs
- improved scanned-tab title / URL spacing
- visible workspace tab state badges
- recovery reuse logic for tabs already reopened from Recovery Journal
- diagnostics tracing for Add Current Active Tab

## Tab State Badges

Workspace tab cards now show badges for:

- Record ID
- Open
- Missing
- Ambiguous
- Grouped
- Ungrouped
- Unassigned
- match status

These badges are not final UI design. They are functional state indicators for validation and later AI contract design.

## Recovery Reuse Rule

If a closed-tab recovery event has already used Reopen URL, then Re-add to Workspace should reuse that reopened browser tab when possible instead of opening a second duplicate browser tab.

The expected event metadata is:

```text
browserTabReused: true
restoreMode: readd_reused_reopened_url
```

If no previously reopened tab is found, Re-add to Workspace keeps the previous behavior and opens the saved URL.

## AI Contract Relevance

This closure pass prepares the tab layer for AI command proposals such as:

- add the current tab
- select these scanned tabs
- add selected tabs
- group workspace tabs
- focus this role group
- refresh workspace tab metadata
- recover this tab
- clear workspace tab records

The AI layer should propose these actions through the deterministic surface rather than directly manipulating Chrome APIs.
