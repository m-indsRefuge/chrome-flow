# V0.2K Patch — Stable Workspace Tab Record IDs

## Purpose

This patch fixes the second-order duplicate-tab issue exposed by V0.2K.

V0.2K allowed multiple browser tab instances with the same URL to enter the workspace. That was correct. However, older workspace controls still targeted tab records using `tabKey`, which is derived from URL and title and is therefore not unique.

The result was that role assignment, focus, close, and remove actions could target the first matching same-URL record instead of the specific visible card the user interacted with.

## Identity Model

Chrome Flow now distinguishes:

```text
workspaceTabId = Chrome Flow's stable saved tab record identity
tabId = Chrome's live browser tab identity
tabKey = page/title fingerprint, useful as metadata but not unique
url = page identity, useful as metadata but not unique
```

## Behavior

Done when:

- Existing workspace tabs are migrated to include `workspaceTabId`.
- New identity-aware controls target workspace records by `workspaceTabId`.
- Alias updates use `workspaceTabId`.
- Role updates use `workspaceTabId`.
- Focus Tab uses `workspaceTabId` first, then exact Chrome tab ID.
- Close Browser Tab uses `workspaceTabId`.
- Remove from Workspace uses `workspaceTabId`.
- Focus Group resolves live tabs through stable records instead of broad URL matching.
- Same-title / same-URL duplicate workspace records can be assigned to different roles independently.
- Same-title / same-URL duplicate workspace records can be focused independently when their live Chrome tab IDs still exist.
- Ambiguous fallback matching refuses to guess when multiple live same-URL candidates exist.

## Implementation

The fix is implemented as a side panel module:

```text
src/sidepanel/workspace-tab-record-ids.js
```

The module:

- migrates records missing `workspaceTabId`
- annotates rendered workspace tab controls with `data-workspace-tab-id`
- intercepts role, alias, focus, close, remove, and group-focus actions in capture phase
- records diagnostics for stable-record actions

## Known Limit

This patch stabilizes the visible workspace tab controls. Some older background metadata routines may still use broad URL fallback internally until the main sidepanel controller is fully refactored. The critical user-facing duplicate-role/focus failure is handled by this patch.
