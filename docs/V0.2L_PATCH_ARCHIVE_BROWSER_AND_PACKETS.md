# V0.2L Patch — Archive Browser and Packaged Workspace Snapshots

## Purpose

This patch extends Workspace Session Control so archived workspaces can be inspected and packaged directly from the side panel.

The first V0.2L slice could archive the current workspace and start fresh, but the archive store only surfaced as a count. The active workspace snapshot included archive summaries but did not let the Operator choose a specific archived workspace to package.

## Behavior

Done when:

- Workspace Session Control shows an Archived Workspaces selector.
- The selector lists archived workspaces by archive name, tab count, and system event count.
- Selecting an archive displays a compact summary.
- The Operator can copy an active workspace packet.
- The Operator can copy a selected archived workspace packet.
- Active workspace packets and archived workspace packets include schema metadata similar to diagnostic packets.
- Packet notes warn that titles, URLs, notes, system events, and recovery data may be included.

## Packet Schemas

Active workspace packet:

```text
workspace-packet-v0.2
```

Archived workspace packet:

```text
archived-workspace-packet-v0.1
```

## Current Non-Goals

This patch does not restore archived workspaces into the active workspace slot.

Restore is intentionally deferred because it can overwrite or merge active state and needs a separate safety design.
