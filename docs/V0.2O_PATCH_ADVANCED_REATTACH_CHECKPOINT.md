# V0.2O Patch — Advanced Controls Reattach Checkpoint

## Purpose

This checkpoint reattaches the refined Advanced Tab Controls layer above the restored and validated base tab-management runtime.

The goal is to validate that the advanced layer can operate without interrupting the base layer.

## Runtime Load Order

The side panel now loads:

```text
diagnostics.js
sidepanel.js
workspace-session-control.js
search-workspace-intake.js
advanced-tab-controls.js
```

Stylesheets:

```text
sidepanel.css
advanced-tab-controls.css
```

## Base Layer Protected Behaviors

Before testing advanced controls, confirm these still work:

- selected scanned-tab intake
- scanned tabs clear after selected add
- search auto-intake
- role assignment
- native Chrome grouping
- focus tab
- focus group
- workspace packet export
- diagnostic packet export

## Advanced Layer Reattached Behaviors

The reattached advanced layer should expose:

- Collapse Workspace Chrome Groups
- Expand Workspace Chrome Groups
- Move Workspace Into New Window
- Arrange Tabs by Role Order
- Reopen All Missing Tabs
- Copy Workspace URL List
- Refresh Duplicate URL Review
- Duplicate URL Review

The following remain intentionally absent:

- Pin Workspace Tabs
- Unpin Workspace Tabs
- Move Workspace Tabs Into One Window

## Validation Order

Use this order so regressions are easy to isolate:

```text
1. Reload extension.
2. Confirm side panel loads.
3. Confirm Advanced Tab Controls appears.
4. Run a short base selected-tab intake test.
5. Run a short search auto-intake test.
6. Run Chrome grouping.
7. Test collapse and expand groups.
8. Test arrange by role order.
9. Test Move Workspace Into New Window.
10. Refresh Workspace Tab Metadata.
11. Recreate Chrome groups if group IDs become stale after window movement.
12. Copy workspace and diagnostic packets.
```

## Expected Evidence

A clean pass should show:

- `selected_tabs_added`
- `browser_search_tab_added_to_workspace`
- `chrome_tab_groups_created`
- `chrome_tab_groups_collapsed`
- `chrome_tab_groups_expanded`
- `workspace_tabs_arranged_by_role`
- `workspace_tabs_moved_to_new_window`
- `pendingActionTraces: []`

## Integration Note

Do not integrate the advanced controls natively until this reattached layer validates cleanly. Native integration should happen only after the advanced layer is stable above the base layer.
