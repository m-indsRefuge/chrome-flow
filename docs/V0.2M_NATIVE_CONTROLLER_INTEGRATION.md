# V0.2M Patch — Native Controller Integration

## Purpose

This patch converts the validated V0.2K–V0.2M hardening behavior from override modules into native `sidepanel.js` controller behavior.

The temporary override modules were useful during active bug isolation because they allowed small surgical fixes. Once the behavior was validated, keeping capture-phase modules in front of older controller behavior became an architectural risk.

## What Changed

The following behaviors are now native to `src/sidepanel/sidepanel.js`:

- instance-aware tab intake
- same-URL duplicate awareness
- stable `workspaceTabId` migration
- alias and role updates by `workspaceTabId`
- focus / close / remove by `workspaceTabId`
- combined Recovery Journal Re-add + browser reopen for closed tabs
- stable one-to-one browser tab resolution
- native Chrome grouping through stable resolution
- workspace metadata refresh through stable resolution
- tab-status refresh through stable resolution
- recovery group recreation using the native grouping path

## Removed Runtime Override Modules

The following runtime modules were removed after their behavior was integrated into the native controller:

```text
src/sidepanel/intake-instance-matching.js
src/sidepanel/workspace-tab-record-ids.js
src/sidepanel/recovery-combined-restore.js
src/sidepanel/stable-browser-tab-resolution.js
```

## Current Runtime Script Path

The side panel now loads:

```text
src/sidepanel/diagnostics.js
src/sidepanel/sidepanel.js
src/sidepanel/workspace-session-control.js
```

`diagnostics.js` remains separate because it is a testing/observability surface.

`workspace-session-control.js` remains separate because it dynamically adds the archive/start-fresh workspace control surface and is not an override for old tab behavior.

## Validation Intent

After this refactor, validation should confirm that behavior remains the same as the previous successful V0.2M validation, but diagnostic packets should no longer show behavior as coming from a separate override script path.
