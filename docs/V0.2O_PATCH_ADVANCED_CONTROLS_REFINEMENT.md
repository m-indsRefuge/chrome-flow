# V0.2O Patch — Advanced Controls Refinement

## Purpose

This patch refines the V0.2O advanced deterministic tab controls after validation surfaced two product-level issues:

1. Workspace-level pin/unpin conflicts with Chrome Flow's grouping model.
2. Move Workspace Tabs Into One Window could appear to do nothing when the target was already the current window.

The patch keeps Advanced Tab Controls separate from the native base controller until this refined advanced layer validates cleanly.

## Removed

The following controls were removed from the Advanced Tab Controls surface:

```text
Pin Workspace Tabs
Unpin Workspace Tabs
```

Reason:

Pinned tabs are pulled into Chrome's pinned-tab browser area, which fights against the workspace role-grouping model. Pinning may return later as a smaller browser utility such as "Pin Current Tab," but it should not be part of the workspace grouping surface.

## Replaced

The previous control:

```text
Move Workspace Tabs Into One Window
```

was replaced with:

```text
Move Workspace Into New Window
```

## New Window Behavior

Move Workspace Into New Window now:

- resolves open workspace tabs with the stable one-to-one resolver
- creates a new Chrome window using the first resolved workspace tab
- moves the remaining resolved workspace tabs into that new window
- focuses the new window
- refreshes workspace tab metadata so new `windowId` values are persisted
- preserves Chrome Flow workspace state through `chrome.storage.local`
- records `workspace_tabs_moved_to_new_window`

Important note:

Moving tabs into a new Chrome window may disrupt native Chrome group IDs. If group badges appear stale after the move, run Create Chrome Tab Groups again.

## Missing Tab Clarification

The UI and System Journal now clarify the missing-tab rule:

```text
Missing tab = workspace record still exists, but live browser tab is gone.
Removed tab = workspace record was removed and should be restored from Recovery Journal.
```

To test Reopen All Missing Tabs correctly:

1. Add a tab to the workspace.
2. Close that browser tab directly in Chrome, not through Chrome Flow.
3. Click Refresh Workspace Tab Metadata.
4. Confirm the workspace record becomes Missing.
5. Click Reopen All Missing Tabs.

If a tab was closed through Chrome Flow, it is intentionally removed from workspace records and should be restored through Recovery Journal instead.

## Current Advanced Runtime Surface

Advanced Tab Controls now includes:

- Collapse Workspace Chrome Groups
- Expand Workspace Chrome Groups
- Move Workspace Into New Window
- Arrange Tabs by Role Order
- Reopen All Missing Tabs
- Copy Workspace URL List
- Refresh Duplicate URL Review
- Duplicate URL Review surface

## Integration Plan

Do not integrate the advanced layer into the native base controller yet.

Recommended sequence:

```text
1. Validate this V0.2O refinement patch.
2. Confirm the accepted advanced controls are stable.
3. Remove or adjust any remaining problematic actions.
4. Then integrate accepted advanced controls into native base code.
```
