# V0.2M — Stable Browser Tab Resolution for Grouping and Metadata Refresh

## Purpose

This slice fixes the remaining duplicate-tab identity issue discovered after V0.2K and V0.2L validation.

The user-facing controls were already using stable `workspaceTabId` values, but native Chrome grouping and metadata refresh still depended on older broad URL / tabKey matching. That meant two workspace records with the same URL could collapse onto one live Chrome `tabId` during grouping or refresh.

## Identity Rule

Chrome Flow now treats browser tab resolution as a one-to-one assignment problem:

```text
one workspaceTabId -> zero or one live Chrome tab
one live Chrome tab -> zero or one workspace record
```

A live browser tab may not be consumed by more than one workspace record during grouping, metadata refresh, or tab-status refresh.

## Resolution Order

For each workspace tab record:

1. Match by exact saved Chrome `tabId` if that live tab exists and has not already been consumed.
2. If the saved exact `tabId` was already consumed by another workspace record, attempt a single unconsumed same-URL repair.
3. Use same-URL fallback only if exactly one unconsumed live tab has that URL.
4. Use same-tabKey fallback only if exactly one unconsumed live tab has that tabKey.
5. If multiple fallback candidates exist, mark the record ambiguous instead of guessing.
6. If no candidate exists, mark the record missing.

## Intercepted Actions

The stable resolver now intercepts:

- Create Chrome Tab Groups
- Refresh Workspace Tab Metadata
- Refresh Tab Status

These actions now write their own diagnostics because they must stop the older handlers from running.

## Diagnostics

V0.2M diagnostic metadata includes:

- `resolutionMode: stable_one_to_one`
- `workspaceTabId`
- saved tab ID
- saved URL
- matched live tab ID
- live window ID
- live group ID
- match status
- candidate count

## Expected Result

Same-URL duplicate workspace records should no longer be overwritten with the same live Chrome `tabId` during grouping or metadata refresh.

If two same-URL tabs cannot be safely distinguished, Chrome Flow should mark the record as ambiguous rather than silently choosing the wrong live tab.
