# V0.2O Patch — New Window Stability Override

## Purpose

This patch stabilizes `Move Workspace Into New Window` after validation showed two different browser-level issues:

1. The original tab-based window creation did create a new window, but it could appear minimized or lose focus.
2. The later empty-window strategy could visibly open a new window, pause, then collapse after temporary-tab cleanup.

## Current Strategy

A small advanced-layer override is loaded after `advanced-tab-controls.js`:

```text
advanced-new-window-stability.js
```

It intercepts only the `Move Workspace Into New Window` button and prevents the older advanced handler from running.

## Runtime Load Order

```text
diagnostics.js
sidepanel.js
workspace-session-control.js
search-workspace-intake.js
advanced-tab-controls.js
advanced-new-window-stability.js
```

## Stable Move Behavior

The override uses the earlier tab-based strategy that successfully created the new window, but adds focus recovery:

```text
1. Resolve open workspace tabs with stable one-to-one matching.
2. Sort by workspace role order.
3. Create a new Chrome window from the primary workspace tab.
4. Force the new window to state: normal and focused.
5. Move remaining workspace tabs into the new window.
6. Force the new window to normal/focused again.
7. Refresh workspace metadata.
8. Recreate role-based Chrome tab groups in the new window.
9. Force focus recovery again.
10. Refresh workspace metadata again.
```

## Expected Event Evidence

The `workspace_tabs_moved_to_new_window` event should include:

```text
newWindowCreationMode: primary_tab_new_window_focus_recovery
finalWindow: {
  state: "normal",
  focused: true
}
recreatedChromeGroups: true
recreatedGroupCount: <number>
groupedTabCount: <number>
```

## Validation Checklist

1. Reload the extension.
2. Confirm Advanced Tab Controls appears.
3. Add 3–5 tabs to the workspace.
4. Assign roles.
5. Create Chrome Tab Groups.
6. Click `Move Workspace Into New Window` once.
7. Confirm the new window persists instead of minimizing or closing.
8. Confirm workspace tabs are in that new window.
9. Confirm role groups are recreated.
10. Copy workspace and diagnostic packets.

## Integration Note

This is a temporary advanced-layer stability override. After validation, consolidate accepted behavior into the advanced module, then later into the native controller during a deliberate native-controller rewrite pass.
