# V0.2O Patch — New Window Stability V2

## Purpose

This patch replaces the first new-window stability override with a v2 override.

Validation showed the previous override successfully moved tabs into a new window, but failed during group recreation because the local helper `groupBy` was missing.

Observed error:

```text
ReferenceError: groupBy is not defined
```

The failure happened after workspace tabs moved, so the browser state changed but Chrome Flow could not complete role-based group recreation.

## Runtime Load Order

The side panel now loads:

```text
diagnostics.js
sidepanel.js
workspace-session-control.js
search-workspace-intake.js
advanced-tab-controls.js
advanced-new-window-stability-v2.js
```

The earlier `advanced-new-window-stability.js` file remains in the repo for now but is no longer loaded.

## V2 Behavior

The v2 override:

- intercepts only `Move Workspace Into New Window`
- uses the primary-tab new-window strategy that visibly creates the new window
- forces the new window to normal/focused state
- moves remaining workspace tabs into the new window
- refreshes workspace metadata
- recreates role-based Chrome groups in the target new window
- includes its own local `groupBy` helper
- records `primary_tab_new_window_focus_recovery_v2`

## Expected Evidence

A clean event should include:

```text
workspace_tabs_moved_to_new_window
newWindowCreationMode: primary_tab_new_window_focus_recovery_v2
recreatedChromeGroups: true
recreatedGroupCount: <number>
groupedTabCount: <number>
finalWindow: { state: "normal", ... }
```

## Validation Checklist

1. Pull latest and reload the extension.
2. Start with 3–5 workspace tabs.
3. Assign roles.
4. Click `Create Chrome Tab Groups`.
5. Confirm the role group header controls appear where expected.
6. Click `Move Workspace Into New Window` once.
7. Confirm the new window persists.
8. Confirm the tabs move into the new window.
9. Confirm Chrome groups are recreated in the new window.
10. Click `Refresh Workspace Tab Metadata`.
11. Copy workspace and diagnostic packets.

## Note on Group Header Controls

`Focus Group` and `Remove Chrome Group` are group-level controls. They are expected on the rendered role/group container, not inside every individual tab card.

If they are still missing after v2 successfully recreates groups, inspect the native group-rendering path separately.
