# V0.2K Patch — Stable Workspace Tab Record IDs Checklist

## Migration validation

1. Pull latest repo and reload Chrome Flow.
2. Open the side panel.
3. Confirm existing workspace tabs render normally.
4. Confirm each workspace tab card shows a small Record badge.
5. Open Developer Diagnostics and confirm there are no runtime errors.
6. Copy a diagnostic packet if migration issues appear.

## Duplicate tab role validation

1. Clear workspace tabs.
2. Open two different Chrome tabs to the same URL.
3. Scan Current Window Tabs.
4. Select both same-URL tabs.
5. Click Add Selected Tabs to Workspace.
6. Confirm both tab instances appear in Workspace Tabs.
7. Assign the first duplicate tab to one role.
8. Assign the second duplicate tab to a different role.
9. Confirm each duplicate moves to its own selected role subgroup.
10. Confirm no duplicate remains unassigned unless intentionally left unassigned.
11. Confirm System Journal records role updates for each action.
12. Confirm Developer Diagnostics records workspace_tab_role_updated with different workspaceTabId values.

## Duplicate tab focus validation

1. With two same-URL duplicate tabs still open, click Focus Tab on the first duplicate workspace card.
2. Confirm Chrome focuses the corresponding live browser tab.
3. Click Focus Tab on the second duplicate workspace card.
4. Confirm Chrome focuses the other corresponding live browser tab.
5. Confirm System Journal includes browser_tab_focused events with workspaceTabId metadata.
6. Confirm diagnostics include workspace_tab_focused_by_record_id events.

## Duplicate tab remove/close validation

1. Remove one duplicate from Workspace.
2. Confirm only that visible card is removed.
3. Confirm the other duplicate remains in the workspace.
4. Close one duplicate through Chrome Flow.
5. Confirm only that selected duplicate is closed and removed.
6. Confirm Recovery Journal contains the correct snapshot.
