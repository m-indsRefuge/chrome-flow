# V0.2K Patch — Recovery Re-add Reopens Closed Tabs Checklist

## Closed-tab recovery validation

1. Pull latest repo and reload Chrome Flow.
2. Add a browser tab to the workspace.
3. Assign it a role.
4. Click Close Browser Tab from Chrome Flow.
5. Provide a reason.
6. Confirm the browser tab closes and the workspace record is removed.
7. Open Journal → Recovery Journal.
8. Find the closed-tab recovery card.
9. Click Re-add to Workspace.
10. Confirm the saved URL opens as a browser tab.
11. Confirm the workspace record is restored.
12. Confirm the restored record keeps its role.
13. Confirm System Journal records workspace_tab_readded.
14. Confirm the workspace_tab_readded event includes browserTabReopened: true.
15. Confirm Developer Diagnostics resolves Re-add to Workspace as action_success.

## Removed-tab recovery validation

1. Add a browser tab to the workspace.
2. Click Remove from Workspace without closing the browser tab.
3. Provide a reason.
4. Open Recovery Journal.
5. Click Re-add to Workspace.
6. Confirm the workspace record is restored.
7. Confirm a duplicate browser tab is not opened just because the workspace record was removed.
8. Confirm System Journal records workspace_tab_readded.

## Repeated re-add validation

1. Click Re-add to Workspace again for the same recovery card.
2. Confirm Chrome Flow does not create a duplicate workspace record.
3. Confirm Chrome Flow records a terminal workspace_tab_readded event.
4. Confirm diagnostics does not produce action_no_result_observed for this repeated Re-add attempt.
