# V0.2O Patch — Advanced Controls Refinement Checklist

## Load validation

1. Pull latest repo and reload Chrome Flow.
2. Confirm Advanced Tab Controls appears.
3. Confirm Pin Workspace Tabs is gone.
4. Confirm Unpin Workspace Tabs is gone.
5. Confirm Move Workspace Into New Window appears.
6. Confirm Move Workspace Tabs Into One Window is gone.
7. Confirm the missing-tab explanation appears under the Advanced Tab Controls status area.

## New-window workspace validation

1. Add several tabs to the workspace.
2. Assign roles.
3. Create Chrome Tab Groups.
4. Click Move Workspace Into New Window.
5. Confirm a new Chrome window opens.
6. Confirm all open resolved workspace tabs move into that new window.
7. Confirm no workspace records are lost.
8. Confirm aliases and roles remain intact.
9. Confirm the side panel still sees the same active workspace state.
10. Click Refresh Workspace Tab Metadata.
11. Confirm workspace tab records now show the new windowId values in the workspace packet.
12. Confirm System Journal records workspace_tabs_moved_to_new_window.
13. Confirm the event includes newWindowId, tabIds, workspaceTabIds, and resolutionMode: stable_one_to_one.
14. If Chrome group badges look stale, click Create Chrome Tab Groups again and confirm grouping recovers.

## Group collapse / expand regression validation

1. Create Chrome Tab Groups.
2. Click Collapse Workspace Chrome Groups.
3. Confirm groups collapse.
4. Click Expand Workspace Chrome Groups.
5. Confirm groups expand.
6. Confirm System Journal records chrome_tab_groups_collapsed and chrome_tab_groups_expanded.

## Arrange by role regression validation

1. Assign several roles.
2. Click Arrange Tabs by Role Order.
3. Confirm tabs arrange within their current window by workspace role order.
4. Confirm System Journal records workspace_tabs_arranged_by_role.

## Missing-tab behavior validation

1. Add a tab to the workspace.
2. Close that browser tab directly in Chrome, not through Chrome Flow.
3. Click Refresh Workspace Tab Metadata.
4. Confirm the workspace record appears as Missing.
5. Click Reopen All Missing Tabs.
6. Confirm the missing tab reopens.
7. Confirm System Journal records missing_workspace_tabs_reopened.
8. Confirm ambiguous tabs are skipped rather than guessed.

## Removed-through-Chrome-Flow distinction

1. Add a tab to the workspace.
2. Close it through Chrome Flow using Close Browser Tab.
3. Confirm it appears in Recovery Journal.
4. Click Reopen All Missing Tabs.
5. Confirm it does not reopen through missing-tab recovery because the workspace record was intentionally removed.
6. Restore it through Recovery Journal instead.

## Export and duplicate review regression validation

1. Click Copy Workspace URL List.
2. Paste into a temporary text file and confirm Markdown grouped by role.
3. Add two same-URL tabs to the workspace.
4. Click Refresh Duplicate URL Review.
5. Confirm duplicate group appears.
6. Apply alias, focus tab, and remove one duplicate record.
7. Confirm browser tab remains open after duplicate record removal.
8. Confirm Recovery Journal includes a restore-capable event.

## Packet validation

1. Copy Active Workspace Packet.
2. Copy Diagnostic Packet.
3. Confirm no runtime errors.
4. Confirm pendingActionTraces is empty after all traced actions complete.
5. Paste both packets into the build thread for review.
