# V0.2M Stable Browser Tab Resolution Checklist

## Setup

1. Pull latest repo and reload Chrome Flow.
2. Start from a clean or low-noise workspace.
3. Open two different Chrome tabs with the same URL.
4. Add both same-URL tabs to the workspace.
5. Assign each same-URL tab a different alias and role.

## Metadata refresh validation

1. Click Refresh Workspace Tab Metadata.
2. Confirm the action succeeds.
3. Copy Active Workspace Packet.
4. Confirm each duplicate workspace record keeps a distinct `workspaceTabId`.
5. Confirm each duplicate workspace record keeps a distinct live `tabId` when both live browser tabs still exist.
6. Confirm the timeline event includes `resolutionMode: stable_one_to_one`.
7. Confirm the event includes `resolutionResults` with match statuses.

## Native grouping validation

1. Click Create Chrome Tab Groups.
2. Confirm the action succeeds.
3. Confirm all open assigned workspace tabs are grouped.
4. Copy Diagnostic Packet.
5. Confirm action_success for Create Chrome Tab Groups includes `resolutionMode: stable_one_to_one`.
6. Confirm diagnostics include `stable_chrome_grouping_completed`.
7. Copy Active Workspace Packet.
8. Confirm same-URL duplicate records do not collapse onto the same live `tabId`.
9. Confirm same-URL duplicate records keep their separate roles.

## Tab-status validation

1. Click Refresh Tab Status.
2. Confirm the action succeeds.
3. Confirm System Journal records `workspace_tab_status_refreshed`.
4. Confirm the event includes `resolutionMode: stable_one_to_one`.
5. Confirm the UI counters match the workspace state.

## Ambiguity safety validation

1. Create a condition where a workspace record cannot be safely matched to one exact live tab.
2. Click Refresh Workspace Tab Metadata.
3. Confirm Chrome Flow marks the record missing or ambiguous instead of assigning it to the wrong live tab.
4. Confirm diagnostics include a match status such as `ambiguous_url_matches`, `ambiguous_tab_key_matches`, or `exact_tab_id_consumed`.
