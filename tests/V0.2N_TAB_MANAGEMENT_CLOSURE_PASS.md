# V0.2N Tab Management Closure Pass Checklist

## Load validation

1. Pull latest repo and reload Chrome Flow.
2. Confirm the side panel loads without runtime errors.
3. Confirm Workspace Session Control still appears.
4. Confirm Developer Diagnostics still appears.

## Intake control validation

1. Confirm Workspace Intake now shows:
   - Scan Current Window Tabs
   - Add Current Active Tab
   - Select All Scanned Tabs
   - Deselect All Scanned Tabs
   - Add Selected Tabs to Workspace
   - Clear Scanned Tabs
2. Click Scan Current Window Tabs.
3. Confirm scanned tab cards show title, URL, and tab metadata with clear spacing.
4. Click Select All Scanned Tabs.
5. Confirm all selectable scanned tabs are checked.
6. Click Deselect All Scanned Tabs.
7. Confirm all scanned tabs are unchecked.
8. Select a few tabs and click Add Selected Tabs to Workspace.
9. Confirm scanned tab capture clears after add.
10. Confirm System Journal records selected_tabs_added with intakeMatchingMode: instance_aware.

## Add active tab validation

1. Focus a browser tab that is not in the workspace.
2. Click Add Current Active Tab.
3. Confirm the active tab appears in Workspace Tabs.
4. Confirm System Journal records active_tab_added.
5. Click Add Current Active Tab again while the same tab is active.
6. Confirm it does not create a duplicate workspace record.
7. Confirm System Journal records active_tab_add_skipped.
8. Confirm Developer Diagnostics resolves the action as success or skipped.

## State badge validation

1. Assign aliases and roles to workspace tabs.
2. Confirm each workspace tab card shows state badges.
3. Confirm open tabs show Open.
4. Confirm ungrouped open tabs show Ungrouped.
5. Click Create Chrome Tab Groups.
6. Confirm grouped tabs show Grouped.
7. Close a browser tab outside or through Chrome Flow.
8. Refresh Workspace Tab Metadata.
9. Confirm missing or ambiguous records show an appropriate badge.

## Recovery reuse validation

1. Add a tab to the workspace.
2. Close it through Chrome Flow and provide a reason.
3. Open Recovery Journal.
4. Click Reopen URL.
5. Confirm the URL opens as a browser tab.
6. Click Re-add to Workspace for the same recovery card.
7. Confirm Chrome Flow reuses the browser tab that was already reopened instead of opening a second duplicate.
8. Confirm System Journal records workspace_tab_readded.
9. Confirm the event includes browserTabReused: true and restoreMode: readd_reused_reopened_url.

## Regression validation

1. Focus individual tabs.
2. Focus role groups.
3. Remove one tab from workspace.
4. Close one browser tab from workspace.
5. Re-add from Recovery Journal.
6. Create Chrome Tab Groups.
7. Remove All Chrome Tab Groups.
8. Refresh Workspace Tab Metadata.
9. Refresh Tab Status.
10. Copy Active Workspace Packet.
11. Copy Diagnostic Packet.
12. Confirm pendingActionTraces is empty after actions complete.
