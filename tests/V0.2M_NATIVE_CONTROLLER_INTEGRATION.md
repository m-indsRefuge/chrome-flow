# V0.2M Native Controller Integration Checklist

## Load-path validation

1. Pull latest repo and reload Chrome Flow.
2. Open the side panel.
3. Confirm the side panel loads without runtime errors.
4. Confirm Developer Diagnostics loads.
5. Confirm Workspace Session Control loads.
6. Confirm the previous override scripts are not loaded by `sidepanel.html`.

## Intake validation

1. Start a clean workspace.
2. Scan current window tabs.
3. Add selected tabs to the workspace.
4. Confirm scanned tabs clear after add.
5. Open two different tabs with the same URL.
6. Confirm same-URL duplicate tabs remain selectable unless they are the exact same browser tab instance.
7. Confirm timeline records `selected_tabs_added` with `intakeMatchingMode: instance_aware`.

## Stable record validation

1. Assign aliases and roles to multiple workspace tabs.
2. Confirm role changes target the visible card.
3. Confirm System Journal role events include `workspaceTabId`.
4. Focus a tab from its card.
5. Confirm focus targets the correct live Chrome tab.
6. Close a tab from its card and confirm Recovery Journal receives a snapshot.

## Recovery validation

1. Use Recovery Journal → Re-add to Workspace on a closed-tab event.
2. Confirm the saved URL reopens as a browser tab.
3. Confirm the workspace record is restored with the original `workspaceTabId`.
4. Confirm System Journal records `workspace_tab_readded` with `browserTabReopened: true`.

## Stable grouping and refresh validation

1. Create native Chrome tab groups.
2. Confirm all open assigned workspace tabs are grouped.
3. Confirm System Journal records `chrome_tab_groups_created` with `resolutionMode: stable_one_to_one`.
4. Refresh Workspace Tab Metadata.
5. Confirm System Journal records `workspace_tabs_refreshed` with `resolutionMode: stable_one_to_one`.
6. Confirm duplicate same-URL records do not collapse onto the same live `tabId`.
7. Refresh Tab Status.
8. Confirm System Journal records `workspace_tab_status_refreshed` with `resolutionMode: stable_one_to_one`.

## Regression validation

1. Add a User Journal entry.
2. Copy Active Workspace Packet.
3. Copy Diagnostic Packet.
4. Confirm no pending action traces remain after actions complete.
